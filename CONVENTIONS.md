# 知识库规范

所有笔记必须遵循以下标准，保持一致性。

## 命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 每日笔记 | `YYYY-MM-DD.md` | `2026-06-23.md` |
| 主题文件 | 中文名称 `.md` | `穿搭美学.md` |
| 主题索引 | 固定为 `README.md` | `Topics/金融/README.md` |
| 收件箱 | 无固定格式，归档后删除 | `想法-xxx.md` |

## 文件结构

### 每日笔记

```markdown
# YYYY-MM-DD

## 今日概要

- **主题名**：一句话摘要。
  - 子项说明（如有）

## 读到/看到的

## 下一步计划

## 关联主题

- [[Topics/主题/README|主题名]]

## Tags
#daily
```

### 主题笔记

```markdown
# 主题名称

一句话说明该主题的定位与范围。

## 核心内容

（按章节/模块组织，使用二级标题）

## 相关笔记

- [[Daily/YYYY-MM-DD]] — 事件描述
```

### 收件箱笔记

```markdown
# 标题

内容（随意格式，后续归档时整理）
```

## 标签规范

| 标签 | 用途 |
|------|------|
| `#daily` | 每日笔记 |
| `#inbox` | 收件箱条目 |
| `#todo` | 待办事项 |
| `#idea` | 创意/想法 |
| `#reference` | 外部资料引用 |

## 写作风格

- **术语优先**：用专业术语归纳，避免口语化
- **简洁明确**：一句话能说清的不写两句
- **结构清晰**：善用标题层级、列表、表格
- **双链关联**：提到其他笔记时用 `[[双链]]` 引用

## 记录规范

- **时间归属**：用户说"今天"时，以用户表达的时间为准记录，不假设系统日期
- **即时确认**：每次记录前确认日期，避免内容归属错误
- **专业归纳**：即使内容来自口语描述，记录时也必须用专业术语重新表述

## 归档流程

```
Inbox → 定期审视 → 归档到 Topics/对应主题
                  → 删除 Inbox 原文件
```

- **频率**：每周至少一次
- **判断标准**：内容是否属于某个已有主题？是否值得保留？

## 维护周期

| 周期 | 动作 |
|------|------|
| 每日 | 写 Daily 笔记，记录当日要点 |
| 每周 | 清空 Inbox，回顾 Daily 笔记 |
| 每月 | 更新 Topic 结构，补充遗漏 |
| 每季 | 审视知识体系，调整方向 |
